Info Gathering

  2 minutes  |    280 words

What do we want to know

Domains and Subdomains

We need to find more subdomains to increase the overall attack surface. Besides, hidden and forgotten subdomains may have old/vulnerable versions of applications or dev versions.

IP ranges

We want to find out as much about our target as possible. Additional IP ranges may lead to discovering other domains and subdomains.

Infrastructure

What technology stacks the target is using?

Virtual Hosts

Virtual hosts are similar to subdomains but indicate that a target is hosting multiple applications on the same web server.

There are two aspects of info gathering:

  • Passive: not directly interact with the target
  • Active: directly interact

Passive Info Gathering

WHOIS

whois is like the “white pages” for domain names. It is a TCP-based transaction-oriented query/response protocol listening on TCP port 43 by default. We use it for quering databases containing domain names, IP addresses, or autonomous systems and provide information services to Internet users.

DNS

DNS is the Internet’s phone book.

We need to familiarize ourselves with infrastructure and identify which hosts are publicly accessible.

$ nslookup paydiant.com
Server:		192.168.1.1
Address:	192.168.1.1#53

Non-authoritative answer:
Name:	paydiant.com
Address: 204.74.99.103

Also, we can specify the nameserver.

$ dig paydiant.com @192.168.1.1

A Records for a Subdomain

$ nslookup -query=A $TARGET
$ dig a $TARGET @192.168.1.1

PTR Records for an IP

$ nslookup -query=PTR 173.0.87.51
Server:		192.168.1.1
Address:	192.168.1.1#53

Non-authoritative answer:
51.87.0.173.in-addr.arpa	name = cloudmonitor30.paypal.com.

Authoritative answers can be found from:
$ dig -x 173.0.87.51 @192.168.1.1

ANY Existing Records

$ nslookup -query=ANY $TARGET
$ dig any $TARGET @8.8.8.8

TXT Records

$ nslookup -query=TXT $TARGET
$ dig txt $TARGET @192.168.1.1

MX Records

$ nslookup -query=MX $TARGET
$ dig mx $TARGET @192.168.1.1
